API Reference¶

This section provides detailed documentation for all functions available in the PyGeoHash library.

Core Functions¶

pygeohash.encode(latitude: float, longitude: float, precision=12) → str[source]¶

Encode a position given in float arguments latitude, longitude to a geohash which will have the character count precision.

pygeohash.encode_strictly(latitude, longitude, precision=12)[source]¶

Encode a position given in float arguments latitude, longitude to a geohash which will have the character count precision. When compared to mid, mid should be included. Provide a separate method for backward compatibility.

pygeohash.decode(geohash: str) → LatLong[source]¶

Decode geohash, returning two float with latitude and longitude containing only relevant digits and with trailing zeroes removed.

pygeohash.decode_exactly(geohash: str) → ExactLatLong[source]¶

Decode the geohash to its exact values, including the error margins of the result. Returns four float values: latitude, longitude, the plus/minus error for latitude (as a positive number) and the plus/minus error for longitude (as a positive number).

Numba-accelerated Functions¶

These functions require Numba and NumPy to be installed and provide faster performance for large-scale operations.

pygeohash.nb_point_encode(latitude: float64, longitude: float64, precision: int8 = 12) → unicode_type[source]¶

Encode a point given by latitude and longitude to a geohash of the specified precision.

pygeohash.nb_point_decode(geohash: str) → LatLong[source]¶

Decode geohash, returning two float with latitude and longitude containing only relevant digits.

pygeohash.nb_vector_encode(latitudes: Array, longitudes: Array, precision: int8 = 12) → Array[source]¶

Encode a vector of points given by latitudes and longitudes to a vector of geohashes of the specified precision. This is not exactly a vectorized version of nb_point_encode, but it is way faster and gets faster as the number of points increase.

pygeohash.nb_vector_decode(geohashes: List[str]) → Tuple[source]¶

Decode geohashes, returning two Arrays of floats with latitudes and longitudes containing only relevant digits. This is not exactly a vectorized version of nb_point_decode, but it is way faster and gets faster as the number of geohashes increase.

pygeohash.nb_decode_exactly(geohash: str) → ExactLatLong[source]¶

Decode the geohash to its exact values, including the error margins of the result. Returns four float values: latitude, longitude, the plus/minus error for latitude (as a positive number) and the plus/minus error for longitude (as a positive number).
